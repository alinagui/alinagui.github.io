Java构造器

C++中最先引入了构造器的概念,这是一个在创建对象时被自动调用的特殊方法.Java中也采用了构造器,并额外提供了"垃圾回收器",在Java中,通过提供构造器,类的设计者可确保每个对象都得到初始化.创建对象是,如果其类具有构造器,Java就会在用户有能力操作对象之前自动调用相应的构造器,从而保证了初始化的进行.

构造器的命名方式:

类名(){};

以下就是一个带有构造器的简单类:

public class FirstTest{

public static void main(String [] args){

for(int i = 0; i < 5; i++){

new Rock();

}

}

}

class Rock{

Rock(){

System.out.print("Rock" + " ");

}

}

Output:

Rock Rock Rock Rock Rock

如上面的代码,在创建对象时:

new Rock();

将会为对象分配储存空间,并调用相应的构造器,这就确保了在你能操作对象之前,它已经被恰当地初始化了.

值得注意的是,由于构造器的名称必须和类型相同,所以"每个方法的首字母必须小写"的编码分格并不适合构造器了

无参数的构造器叫做默认构造器,在如果你没自定义构造器时,java在编译时会自动创建默认构造器.没有它的话,就没有方法可调用,就无法创建对象.但是,如果已经定义了一个构造器(无论是否有参数),编译器就不会帮你自动创建默认构造器
